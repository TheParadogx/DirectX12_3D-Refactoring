### プロジェクトについて
このプロジェクトは技術力向上と研究のために作成しています。
前回フレームワーク規模まで作成したのでリファクタリングかつある程度の効率化を計る目的もあります。

重視する点は、効率と可読性のいい塩梅を目指す点。
拡張性と効率と可読性をある程度両立させたいと思っています


### 命名規則
フィールド：m + 変数名
メソッド　：Pascal

static変数：s + 変数名


### コーディング
現状はスタティックライブラリなので必要ないのですがDLLにする際に
「ENGINE_API」を付けているモノだけしかAPIとしてexeのほうに公開されないので注意。

ENGINE_API：DLL境界をまたぐときのデバックリリースでの分岐用（Libでは意味はない）
いつかできたらいい


### if文
ガード文は基本的に1行で書く。
ロジック判定は拡張する可能性があるものすべて1行IFで書くことを禁止する。



### 初期化の際のルール

クラスはコンストラクタで初期化を行うこと。
定義部分で初期化をしない。


### コメント
基本的にはsummary付けますがシンプルな構造体などは1行コメントで済ませます。
また/**/はインテリセンスが正常に機能する場合で複数行の時は優先します。

### マクロについて
#ifdefではなく#if defined() を使用します。
複合条件ができる上に#if に条件判定を統一できるので可読性が向上するかも。
